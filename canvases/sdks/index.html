Canvas 14 · Product

SDKs & CLIs

Nine distinct surfaces ship the same Sardis API to wherever the agent actually lives — a Python script, a Next.js app, a Claude Desktop chat, an n8n workflow, a Telegram conversation, a checkout page, or a terminal. One auth model (sk_ API keys), one semantic contract, nine idiomatic entry points.

TL;DR The SDK layer is intentionally thin: each binding wraps the same REST surface at api.sardis.sh with types, errors, retries, and pagination. Nothing "clever" happens client-side — policy, mandate, and MPC signing stay server-authoritative. The cleverness lives in how many environments the same API can reach without the developer learning anything new.

Auth

One API-key model across every surface.

Every surface authenticates with a prefixed secret key. Prefix encodes environment and blast radius; require_principal in the API gateway hashes the key (SHA-256), looks up the owning merchant/agent, and attaches a principal to the request for the orchestrator to consume.

PrefixEnvironmentUseRotation
sk_live_…ProductionReal money, real compliance, real rate limits.Hot-rotatable from dashboard; old key decays 24h.
sk_test_…SandboxPolicy & mandate logic end-to-end, but settlement is simulated.Self-serve in dashboard.
sk_dev_…LocalEmitted by sardis auth login against local API.Never leaves laptop.
pk_live_…Public / clientCheckout, spend widget — rate-limited, read-biased scope only.Safe to embed.
01 · Python SDK

sardis-sdk — async httpx client.

Lives at packages/sardis-sdk-python/. Async is the default (AsyncSardisClient); sync exists for scripts. Connection pooling, retry with exponential backoff, token refresh, and typed errors are built-in.

Install & quickstart

# install
pip install sardis-sdk
# or
uv add sardis-sdk

# set API key
export SARDIS_API_KEY=sk_live_…

From packages/sardis-sdk-python/examples/quickstart.py — the canonical flow: create agent → create wallet → parse a natural-language policy → pay → check balance. 

import asyncio, os
from sardis_sdk import AsyncSardisClient

async def main():
    async with AsyncSardisClient(api_key=os.environ["SARDIS_API_KEY"]) as client:
        agent   = await client.agents.create(name="My First Agent")
        wallet  = await client.wallets.create(agent_id=agent.agent_id, chain="base")

        policy  = await client.policies.parse(
            text="Max $100 per transaction, $500 per day, no gambling",
        )
        await client.policies.apply(wallet_id=wallet.wallet_id, policy=policy)

        payment = await client.payments.send(
            wallet_id=wallet.wallet_id,
            to="merchant_demo",
            amount="25.00",
            purpose="API usage - OpenAI",
        )
        print(payment.status, payment.tx_id)

        balance = await client.wallets.get_balance(wallet.wallet_id)
        print(balance.balance, balance.currency)

asyncio.run(main())

Typed error handling

Every error inherits from SardisError — catch the base, or narrow to specifics. Source: packages/sardis-sdk-python/src/sardis_sdk/models/errors.py. 

from sardis_sdk import (
    AsyncSardisClient,
    AuthenticationError,   # 401 — bad or revoked key
    RateLimitError,        # 429 — quota exceeded (respects Retry-After)
    ValidationError,       # 400 — malformed request body
    NotFoundError,         # 404 — unknown wallet / agent / mandate
    TimeoutError,          # request-level timeout
    ConnectionError,       # transport failure before response
    APIError,              # 5xx catchall
)

try:
    await client.payments.send(...)
except RateLimitError as e:
    await asyncio.sleep(e.retry_after)
except AuthenticationError:
    # rotate key and bail
    raise
except APIError as e:
    log.error("sardis %s: %s", e.status_code, e.code)
02 · TypeScript SDK

@sardis/sdk — strict TS, Zod-validated.

Lives at packages/sardis-sdk-js/. Ships three bundles from one source — ESM (dist/index.js), CJS (dist/cjs/index.cjs), and a browser build (dist/browser/index.js). Strict TypeScript, Zod runtime validation at boundaries, interceptors, abort signals, and auto-paginators. 

npm i @sardis/sdk
# or
pnpm add @sardis/sdk
import { SardisClient, RateLimitError, AuthenticationError } from "@sardis/sdk";

const client = new SardisClient({ apiKey: process.env.SARDIS_API_KEY! });

const agent  = await client.agents.create({ name: "My First Agent" });
const wallet = await client.wallets.create({ agent_id: agent.agent_id, chain: "base" });

const policy = await client.policies.parse({
  text: "Max $100 per transaction, $500 per day, no gambling",
});
await client.policies.apply({ wallet_id: wallet.wallet_id, policy });

try {
  const payment = await client.payments.send({
    wallet_id: wallet.wallet_id,
    to: "merchant_demo",
    amount: "25.00",
    purpose: "API usage - OpenAI",
  });
  console.log(payment.status, payment.tx_id);
} catch (err) {
  if (err instanceof RateLimitError) {
    await new Promise(r => setTimeout(r, err.retryAfter * 1000));
  } else if (err instanceof AuthenticationError) {
    throw err;
  }
}

03 · CLI (Python)

sardis-cli — Click-based terminal tool.

Lives at packages/sardis-cli/. One command group per resource, thin wrapper around the Python SDK. Config lives in ~/.sardis/config.toml (commands/auth.py writes it). See commands/ for the full tree. 

# install
pip install sardis-cli

# log in — exchanges email/password or API key for a local token
sardis auth login

# scaffold a new project (API keys, .env, example agent)
sardis init my-agent

# list wallets for the current org
sardis wallet list

# create one
sardis wallet create --agent ag_123 --chain base

# set a natural-language policy
sardis policy set --wallet wal_abc \
  --text "Max $100 per tx, $500/day, no gambling"

# pay
sardis pay --wallet wal_abc --to merchant_demo --amount 25.00 \
  --purpose "OpenAI API usage"

# one-command end-to-end demo (uses sandbox)
sardis demo run

07 · n8n Node

n8n-nodes-sardis — community node.

Lives at packages/n8n-nodes-sardis/. Ships one node (nodes/Sardis/Sardis.node.ts) plus a credential type (credentials/SardisApi.credentials.ts) that securely stores the API key in n8n's encrypted credential store. Requires n8n-workflow ≥ 2.0. 

# Install inside an n8n instance
npm i -g n8n-nodes-sardis

# Or add via n8n Community Nodes UI: Settings → Community Nodes → Install
# Package: n8n-nodes-sardis

Example flow: Gmail → Sardis: Request Approval → Wait → Sardis: Pay. The Sardis node exposes operations for pay, hold, approve, balance, and webhook dispatch so the workflow can stay in one tool.

08 · Spend Widget

sardis-spend-widget — consumer-side approval UI.

Lives at packages/sardis-spend-widget/. An embeddable widget that consumer agent apps drop in so the end user — not the agent — confirms spend above a threshold. Uses a pk_live_… publishable key scoped to a single session; sensitive operations still go through the server. 

<script src="https://widget.sardis.sh/v1/spend.js" async></script>

<sardis-spend
  publishable-key="pk_live_…"
  session-id="ss_live_abc123"
  on-approve="handleApprove"
  on-deny="handleDeny">
</sardis-spend>

The widget subscribes over SSE to spend requests for the session and renders a native-feel approve/deny sheet. Complements the Telegram bot surface for users who live in a browser rather than in chat.

09 · Telegram Bot

sardis-telegram-bot — on-call surface.

Lives at packages/sardis-telegram-bot/. Python bot that mirrors the approval / balance / kill-switch surface into Telegram. Built for solo operators and on-call rotations who want real-time agent-spend pings in a chat they already have open. 

# typical commands (the bot is side-loaded, not published)
/balance              # shows wallet balances across chains
/approvals            # lists pending approvals, with approve/deny buttons
/kill <wallet>     # pauses all outbound spend for a wallet
/status               # last 10 ledger decisions
Shape

Nine entry points, one pipeline.

Whichever surface originates the call, the request hits the same API gateway and the same PaymentOrchestrator. This is what makes "add a new surface" cheap — the trust model doesn't change.
