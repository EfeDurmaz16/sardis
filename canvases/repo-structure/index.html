Canvas 02 · System

Repo Structure

A map of the Sardis monorepo, every top-level folder, what lives inside, and where to look when you're lost. The stack is pragmatic, not prescriptive: Python where the money logic lives, TypeScript where the browser and SDK consumers are, Solidity for the on-chain parts. Nothing here is load-bearing opinion about what language you should be using.

TL;DR Three workspaces (packages/, apps/, contracts/) carry 99% of the production code. Everything else is either supporting material (docs, scripts, demos) or build artefacts. If you're reading code, start in packages/sardis-core. If you're running the public site, start in apps/landing. If you're verifying on-chain, start in contracts/src.

Mental model

How the workspaces fit together.
