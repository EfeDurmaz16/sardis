Canvas 15 · Product

AI Framework Integrations

Sardis is the universal payment layer for every major agent framework. Not "works with a few" — every one that matters today ships a Sardis adapter in packages/, with the same policy engine, the same MPC wallets, and the same ledger underneath. The developer writes agent code; Sardis makes its money verifiable.

19 published adapters 70K+ SDK installs LangChain v1.0.0 MCP-first
Positioning Frameworks come and go; money movement doesn't. By landing in every agent framework's native idiom, Sardis positions itself as the neutral payments substrate — the same way Stripe became neutral for web checkout. Developers don't choose "payments" any more than they choose "HTTPS"; they pick a framework and Sardis is already there.
Shape

Every adapter is the same three-line pattern.

A framework adapter doesn't re-implement Sardis — it translates. Each package imports sardis-sdk (or @sardis/sdk), wraps Sardis primitives (pay, check_balance, set_policy, list_transactions, check_policy) as native tools in that framework's tool protocol, and exposes them as a SardisToolkit. The agent author writes code that looks native; the bytes on the wire are identical across every framework.

Adding a new framework is a one-day job: wrap the five primitive tools in its tool schema, ship as a package, add an example. The policy engine, trust graph, and ledger don't change.

Group 01 · General-purpose orchestration

Where most LLM agents actually live.

Six frameworks. If someone ships an "agent" in 2026, it is almost certainly built on one of these. Each adapter returns a SardisToolkit whose .get_tools() plugs directly into that framework's agent loop.

LangChain
sardis-langchain · v1.0.0
SardisToolkit → five BaseTools (Pay, CheckBalance, SetPolicy, CheckPolicy, ListTransactions) plus a SardisCallbackHandler that logs every tool invocation to the Sardis ledger. Works with any LangChain agent (OpenAI Functions, ReAct, tool-calling). 
pip install sardis-langchain
from sardis_sdk import SardisClient
from sardis_langchain import SardisToolkit

client = SardisClient(api_key="sk_live_…")
tools  = SardisToolkit(client=client, wallet_id="wal_abc").get_tools()
# pass tools to any LangChain agent
Exports: SardisToolkit · SardisCallbackHandler · SardisPayTool · SardisCheckBalanceTool · SardisSetPolicyTool · SardisCheckPolicyTool · SardisListTransactionsTool
CrewAI
sardis-crewai · v1.1.0
Agent payment tools plus pre-built Sardis agents and task templates — "Finance Officer," "Spending Approver" — that drop into a CrewAI crew. Multi-agent crews share a wallet with per-agent policy overlays. 
pip install sardis-crewai
AutoGPT
sardis-autogpt · v2.0.0
AutoGPT blocks that expose Sardis pay, check_balance, and policy ops as first-class workflow nodes. 2.x bumped to match AutoGPT's block architecture refactor. 
pip install sardis-autogpt
OpenAI Agents SDK
sardis-openai-agents · v1.1.0
Sardis tools as Tool instances for OpenAI's Agents SDK — drop into Runner to give any GPT-class model policy-controlled payments. Complementary to sardis-openai (classic function-calling). 
pip install sardis-openai-agents
Claude Agent SDK
sardis-agent-sdk · v1.1.0
Tool definitions + SardisToolHandler for Anthropic's Agent SDK. Covers pay, hold, escrow, subscription, FX quote, policy, balance, transaction list, and mint — a wider surface than most adapters because Claude can handle richer tool schemas. 
pip install sardis-agent-sdk
Exports: SardisToolkit, SardisToolHandler, SARDIS_PAY_TOOL, SARDIS_CHECK_BALANCE_TOOL, SARDIS_CREATE_HOLD_TOOL, SARDIS_CREATE_ESCROW_TOOL, SARDIS_CREATE_SUBSCRIPTION_TOOL, SARDIS_GET_FX_QUOTE_TOOL, SARDIS_SET_POLICY_TOOL, SARDIS_MINT_PAYMENT_TOOL, SARDIS_LIST_TRANSACTIONS_TOOL, SARDIS_CHECK_POLICY_TOOL
Google ADK
sardis-adk · v1.0.0
Wraps Sardis ops as FunctionTools for Google's Agent Development Kit. Ships a create_sardis_agent() helper that returns a pre-configured ADK agent with wallet + policy wired up. 
pip install sardis-adk
Group 02 · Browser & automation

Agents that click, type, and execute shell.

Browser and code-execution agents have the biggest "oops I drained the wallet" risk — they touch live websites and run arbitrary code. Every adapter here force-routes spend through the policy engine before a browser action or shell command can spend money.

Browser Use
sardis-browser-use · v1.1.0
Payment tools for the Browser Use agent. Converts an "agent shops on a real e-commerce site" flow from "trust the DOM" to "Sardis hold + checkout mandate" — actions are reversible until the agent explicitly captures. 
pip install sardis-browser-use
Stagehand / Browserbase
sardis-stagehand
Stagehand actions with a Sardis payment primitive attached. Agents running in Browserbase sessions get the same hold/capture/policy model as any other Sardis agent. TypeScript package. 
npm i sardis-stagehand
Composio
sardis-composio · v1.0.0
Sardis tools in Composio's tool marketplace — so any agent already pulling tools from Composio picks up Sardis without a code change beyond adding the tool name. 
pip install sardis-composio
OpenClaw
sardis-openclaw · v1.0.0
OpenClaw skill — makes Sardis payments a native capability for OpenClaw's computer-use agents. Same five-primitive pattern. 
pip install sardis-openclaw
E2B
sardis-e2b
E2B sandbox template that ships Sardis SDK pre-installed and pre-authenticated to the merchant's sandbox key. Agents that spin up E2B VMs to run code can immediately call sardis.pay(...) without a key-distribution dance. 
e2b template init sardis-e2b
Group 03 · Multi-agent protocols

When agents talk to each other.

Agent-to-agent payments have a protocol problem before they have a payments problem. Sardis ships first-class support for the two multi-agent standards that matter: Google's A2A and Coinbase AgentKit.

Google A2A
sardis-a2a · v1.1.0
Spec-compliant A2A server and client. Agent card discovery at /.well-known/agent.json, JSON-RPC 2.0 task lifecycle, SSE streaming, push notifications. Sardis extends with payment-domain skills (escrow, settlement, trust verification) delivered as DataPart payloads — standards-conformant, payments-aware. 
pip install sardis-a2a
Exports: A2AClient, SardisAgentCard, A2APaymentRequest/Response, AgentDiscoveryService, SSE stream.
Coinbase AgentKit
sardis-agentkit · v0.1.0
Sardis payment actions for Coinbase AgentKit. AgentKit handles on-chain identity; Sardis handles policy and compliance on top — so a CDP-backed agent has non-custodial wallets and natural-language spend rules without rebuilding either. 
pip install sardis-agentkit
Group 04 · Platform

Where agents run inside someone else's product.

Not every agent is a standalone script. These adapters plug Sardis into platforms where non-engineers build agents: Vercel's AI SDK, n8n, Activepieces, ChatGPT custom GPTs.

Vercel AI SDK
@sardis/ai-sdk · v1.1.0
Sardis tools packaged as Vercel AI SDK tool() definitions. Drop into generateText, streamText, or ToolLoopAgent. Ships ESM + CJS + browser bundles; type-safe via Zod. 
npm i @sardis/ai-sdk
MCP Server
@sardis/mcp-server · v1.1.0
50 Sardis tools over Model Context Protocol. Claude Desktop, Cursor, Windsurf, or any MCP host. Simulated mode lets a new user try Sardis from Claude Desktop without an account. See canvas 14 for the full tool list. 
npx @sardis/mcp-server start
n8n
n8n-nodes-sardis · v1.0.0
Community node + credential. Drop Sardis actions (pay, hold, approve, balance, policy check) into any n8n workflow. Credential type stores API keys in n8n's encrypted store. 
npm i -g n8n-nodes-sardis
Activepieces
@sardis/activepieces · v1.0.0
Sardis piece for Activepieces workflow platform. Policy-controlled payments as a drag-and-drop block for non-engineer builders. 
npm i @sardis/activepieces
ChatGPT Custom GPT
sardis-gpt
OpenAPI actions spec + GPT instructions. Gives a ChatGPT custom GPT the ability to pay, check balance, and enforce policies through Sardis — no backend code required beyond hosting the action endpoint (which Sardis provides).
Artifacts: openapi-actions.yaml, instructions.md.
OpenAI function-calling
sardis-openai · v1.0.0
Classic OpenAI function-calling wrapper. Complements sardis-openai-agents for teams still on the bare Chat Completions API. 
pip install sardis-openai
Adapter Anatomy

Every adapter exposes the same five primitives.

The surface area is deliberately tiny. If an agent can do these five things safely, every meaningful agent-payment use case falls out: order an API call, pay a vendor, refill a balance, split a cost between two agents, tighten a policy mid-run.

PrimitiveServer opTypical agent use
pay(to, amount, purpose)POST /v1/payments via PaymentOrchestratorAny outbound payment — API, vendor, user refund.
check_balance(wallet_id)GET /v1/wallets/{id}/balanceDecide whether to spend.
set_policy(text | structured)POST /v1/policies/parse + applySelf-limit before a risky sub-task.
check_policy(intent)POST /v1/policies/checkDry-run: would this pay succeed?
list_transactions(wallet_id)GET /v1/transactionsAgent retrospection / reporting.
Observability

Where integration usage shows up in the dashboard.

Every tool invocation tags its payment with a framework hint (SDK-level X-Sardis-Framework header). The dashboard buckets traffic per framework so operators see where their spend is actually coming from.

Dashboard pageWhat it shows

Release
