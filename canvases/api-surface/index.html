Canvas 04 · System

API Surface

One FastAPI application, 123 routers, grouped by domain. Every router sits behind the same middleware stack (request-id, logging, security headers, rate limit, auth) and — for anything that moves money — funnels into a single PaymentOrchestrator.execute() chokepoint. This canvas is the catalogue.

TL;DR The surface is large on purpose — every money movement, policy decision, compliance check, and webhook callback gets its own router so the seams are explicit. The surface is also uniform on purpose — every request traverses the same middleware, the same principal extractor, and the same pre-execution pipeline. Wide catalogue, narrow spine.

The Spine

Middleware stack — outermost first.

Registered in apps/api/server/main.py. FastAPI/Starlette middleware executes outermost-registered-first on request, innermost-first on response. If you need to add a cross-cutting concern, add it here, not in a router.

01RequestIdMiddlewareAssigns x-request-id, propagates through logs and downstream calls.
02StructuredLoggingMiddlewareJSON log per request: method, path, status, latency_ms, principal, request_id.
03SecurityHeadersMiddlewareCSP, HSTS, X-Frame-Options, Referrer-Policy, Permissions-Policy.
04RequestBodyLimitMiddlewareRejects payloads over configured byte cap — defends against memory-pressure DoS.
05RateLimitMiddlewareRedis-backed token bucket; fail-closed in prod if Redis unreachable (SARDIS_REDIS_URL).
06TapVerificationMiddlewareVerifies Visa TAP agent identity headers (Ed25519 / ECDSA-P256) when enabled.
07X402PaymentMiddlewareImplements x402 Payment Required on gated endpoints; verifies on-chain settlement proof.
08UsageMeteringMiddlewareRecords billable-event rows for Polar; tags by principal + endpoint class.
09ActivityLoggerMiddlewareWrite-ahead of every mutating call into the audit log (feeds Canvas 20).
10CORSMiddlewareCanonical origins: public site origins configured by deployment + merchant embed_origin.

Contract

Error model — RFC 7807 problem+json.

Every non-2xx response conforms to RFC 7807. Clients key off type (stable URI) and code (enum) — not the HTTP status alone, and definitely not the title string.

Shape
Every error carries: type (URI ref), title (short human), status (HTTP code), detail (human detail), code (machine enum), request_id (for grep'ing logs), and when applicable reason_code (policy engine) or retry_after (rate limit).
Categories
400 validation_error
Pydantic schema mismatch — array of field errors in errors[].
401 auth_required
Missing / malformed principal. WWW-Authenticate set.
403 policy_denied
Pre-execution pipeline rejection. reason_code names the gate.
409 conflict
Idempotency collision or state-machine violation.
422 mandate_invalid
AP2 / TAP / mandate chain verification failed.
429 rate_limited
Redis token-bucket refusal. retry_after in seconds.
500 internal
Unclassified — always carries request_id.
503 unavailable
Dependency fail-closed (Redis, KYA, FIDES).
