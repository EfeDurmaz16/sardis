## v0.1.0 - v0.3.0: Foundation (Dec 2025) -- Completed

- Core payment infrastructure
- MPC wallet integration (Turnkey)
- Base chain support
- Basic spending policies
- REST API v2
- MCP Server (4 tools)

## v0.4.0 - v0.5.0: Protocols & Multi-Chain (Jan 2026) -- Completed

- Polygon, Ethereum, Arbitrum, Optimism support
- AP2 (Agent Payment Protocol) implementation
- UCP (Universal Commerce Protocol)
- A2A (Agent-to-Agent) protocol
- TAP identity verification
- MCP Server expanded to 52 tools

## v0.6.0: Fiat Rails & Virtual Cards (Jan 2026) -- Completed

- Fiat on-ramp via Coinbase Onramp
- Fiat off-ramp via Bridge (sandbox/design-partner lane)
- Virtual card issuance (Stripe Issuing)
- Unified balance policy model with quote-based conversion
- KYC integration (Didit production)
- AML screening (Elliptic integration)

## v0.7.0: Database & Persistence (Feb 2026) -- Completed

- PostgreSQL-backed mandate store
- PostgreSQL-backed checkout sessions
- Fireblocks MPC signer integration
- Invoices API
- ABI revert reason decoding

## v0.8.0: Production Hardening (Feb 2026) -- Completed

- Human approval workflows (create/approve/deny/expire)
- Background job scheduler (APScheduler)
- Alembic database migrations
- Wallet freeze capability
- Velocity limit checks for off-ramp
- E2E tests for critical flows
- Prometheus metrics endpoint
- Sentry error tracking
- CI/CD deployment workflows
- Security audit fixes (npm/pip)

## v0.8.1 - v0.8.4: Conformance & Packages (Feb 2026) -- Completed

- AP2 payment modality and agent-presence signals enforced
- TAP algorithm allowlist for message signatures
- Protocol source map for AP2/TAP/UCP/x402
- Deterministic JS bootstrap preflight
- Live-chain conformance gate
- Cloud Run + AWS App Runner deploy automation
- All 19 packages published to npm + PyPI
- Comprehensive security audit (54 fixes across 8 batches)

## v0.8.5 - v0.8.9: Security & Smart Wallets (Feb 2026) -- Completed

- Travel Rule (FATF R.16) compliance
- Card ASA real-time authorization handler
- Redis-backed velocity monitoring
- ERC-4337 smart account contracts + factory + verifying paymaster (Base-first)
- PostgreSQL persistence for all remaining in-memory stores
- Treasury API endpoints (sync, link bank, fund, withdraw, balances)
- USD-first card funding route with stablecoin fallback

## v0.9.0: Multi-Provider Fiat + AI Frameworks (Feb 2026) -- Completed

- Stripe Treasury provider (financial accounts, outbound payments)
- Stripe Issuing provider (virtual cards, real-time auth webhooks)
- Coinbase Onramp (0% USDC fee, smart RampRouter)
- Sub-ledger fiat manager (per-agent balances)
- E2E FiatPaymentOrchestrator with automatic rollback
- OpenClaw skill, sardis-openai, sardis-adk, ChatGPT Actions
- MCP fiat tools (5 new tools)

## v0.9.1 - v0.9.5: Control Plane Hardening (Feb 2026) -- Completed

- A2A trust repository with fail-closed migration guards
- 4-eyes approval enforcement for trust relation mutations
- Secure checkout approval quorum controls
- Card ASA fail-closed defaults in production
- Strict live-mode guardrails on payment execution paths
- Idempotency and replay-proof release gates
- SLO dashboard + PagerDuty alert routing
- DR drill evidence automation

## v0.10.0: ERC-4337 Implementation & Enterprise (Mar 2026) -- Completed

- Immutable hard-limit layer for NL policy parser + fuzz/property tests
- Agent-level sliding window limiter on payment endpoints
- Mainnet gas profiling and optimization with CI gas ceilings
- Turnkey/Fireblocks UserOperation signing path
- Base Sepolia full E2E proof artifact
- Stablecoin-only token allowlist smart contract
- Recurring payments engine
- Multi-tenant organization support
- Advanced analytics dashboard
- Enterprise SLA & support
