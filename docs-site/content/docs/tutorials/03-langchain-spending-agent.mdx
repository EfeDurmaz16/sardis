## What you'll build

A LangChain ReAct agent that can decide to pay for an API call on its own — checking balance, validating against a spending policy, and executing a USDC transfer — all through five purpose-built tools from the `sardis-langchain` toolkit. By the end you'll have an agent loop that takes a natural-language instruction like *"Pay $5 to api.openai.com for credits"* and turns it into a verified, on-chain payment.

## Prerequisites

- Python 3.12+
- `pip install sardis sardis-langchain langchain langchain-openai`
- An OpenAI API key in `OPENAI_API_KEY`
- A Sardis sandbox API key in `SARDIS_API_KEY`

## Step 1: Why a toolkit (and not raw API calls)

You could wire `client.pay.execute` directly into a LangChain `Tool` yourself. People do, and it works for one tool. The reason `SardisToolkit` exists is that a real payment-capable agent needs at least five things — pay, check balance, validate against policy *before* attempting the call, look up the wallet's address book, and read the mandate context — and the right call shape for each is non-obvious. The toolkit gives you all five with consistent argument schemas, agent-friendly descriptions for the LLM router, and a single shared client underneath. The LLM is much better at picking the right tool when each one has a clean single-purpose description.

## Step 2: Instantiate the toolkit

```python
import os
from sardis_langchain import SardisToolkit

toolkit = SardisToolkit(
    api_key=os.environ["SARDIS_API_KEY"],
    agent_id="research-agent",
    base_url="https://sandbox.api.sardis.sh",
)

tools = toolkit.get_tools()
```

The `agent_id` is critical: it scopes every payment to a specific agent identity, so policies, spend logs, and audit trails are partitioned per agent. In production you'll typically mint one `agent_id` per deployed agent role (researcher, shopper, ops bot) rather than per LLM call — this keeps the audit story coherent when the same agent triggers many payments over its lifetime.

`tools` is a list of LangChain `BaseTool` instances. You hand it straight to the agent constructor — no wrapping, no glue code.

## Step 3: Build the agent

```python
from langchain.agents import AgentExecutor, create_react_agent
from langchain_openai import ChatOpenAI
from langchain.prompts import PromptTemplate

prompt = PromptTemplate.from_template("""You are a payments agent. You have these tools:

{tools}

Tool names: {tool_names}

Always check balance with SardisBalanceCheckTool before paying. Refuse if balance is below the requested amount. Reply with the final receipt.

Question: {input}
{agent_scratchpad}""")

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
```

A few decisions worth calling out:

- **`temperature=0`** — for payment agents, you want determinism. A creative agent that occasionally invents merchant names is a liability.
- **The "check balance first" instruction** in the prompt isn't strictly required (Sardis will reject an underfunded payment server-side), but it saves a round-trip and makes the agent's reasoning trace much cleaner to audit later.
- **`verbose=True`** during development — the ReAct trace shows which tool the model picked and why, which is invaluable when debugging weird agent behavior. Turn it off in production or pipe it to a structured log.

## Step 4: Run a payment

```python
result = executor.invoke({
    "input": "Pay $5 in USDC to api.openai.com for API credits."
})
print(result["output"])
```

What you'll see in verbose mode:

1. The model calls `SardisBalanceCheckTool` — confirms there's enough USDC.
2. The model calls `SardisPolicyCheckTool` — confirms the agent's policy allows a $5 payment to `api.openai.com`.
3. The model calls `SardisPayTool` with `merchant="api.openai.com"`, `amount="5.00"`, `token="USDC"`.
4. Sardis settles, returns a receipt, and the model summarizes it in the final answer.

If the policy denies the payment (say, your agent's daily cap is $4), the policy check tool returns the rejection reason and the agent reports it back instead of trying to pay anyway. This is the property you want — *the agent obeys the policy by default*, with the kill switch as a last-resort backstop rather than the only line of defense.

## Step 5: Give the agent a budget

In a real deployment you'd hand the agent a mandate (see [Tutorial 2](/docs/tutorials/02-ap2-mandate-flow)) and let it spend down against the mandate balance. Wire it in like this:

```python
from sardis import Sardis

raw_client = Sardis(
    api_key=os.environ["SARDIS_API_KEY"],
    base_url="https://sandbox.api.sardis.sh",
)
intent = raw_client.payments.create_intent_mandate(
    agent_id="research-agent",
    max_amount_minor=20_000_000,  # $20 budget for this task
    allowed_categories=["ai_api"],
    expires_in_seconds=900,
)

toolkit = SardisToolkit(
    api_key=os.environ["SARDIS_API_KEY"],
    agent_id="research-agent",
    base_url="https://sandbox.api.sardis.sh",
    default_mandate_id=intent["id"],
)
```

Now every payment the agent attempts is scoped to that $20 / 15-minute / `ai_api`-only mandate. The agent can't escape the budget no matter what the prompt asks for — verification happens server-side, before settlement.

## Final: full working example

```python
import os
from sardis import Sardis
from sardis_langchain import SardisToolkit
from langchain.agents import AgentExecutor, create_react_agent
from langchain_openai import ChatOpenAI
from langchain.prompts import PromptTemplate

PROMPT = PromptTemplate.from_template("""You are a payments agent. Tools:

{tools}
Tool names: {tool_names}

Always check balance before paying. Refuse if insufficient.

Question: {input}
{agent_scratchpad}""")

def main() -> None:
    raw = Sardis(
        api_key=os.environ["SARDIS_API_KEY"],
        base_url="https://sandbox.api.sardis.sh",
    )
    intent = raw.payments.create_intent_mandate(
        agent_id="research-agent",
        max_amount_minor=20_000_000,
        allowed_categories=["ai_api"],
        expires_in_seconds=900,
    )

    toolkit = SardisToolkit(
        api_key=os.environ["SARDIS_API_KEY"],
        agent_id="research-agent",
        base_url="https://sandbox.api.sardis.sh",
        default_mandate_id=intent["id"],
    )

    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
    agent = create_react_agent(llm, toolkit.get_tools(), PROMPT)
    executor = AgentExecutor(agent=agent, tools=toolkit.get_tools(), verbose=True)

    out = executor.invoke({
        "input": "Pay $5 in USDC to api.openai.com for API credits."
    })
    print(out["output"])

if __name__ == "__main__":
    main()
```
