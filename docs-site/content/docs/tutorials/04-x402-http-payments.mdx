## What you'll build

A client that calls a paid HTTP endpoint, catches the `402 Payment Required` challenge, settles it through Sardis, and retries the request with the proof-of-payment header — all in a few lines of code. This is the foundation of the agent-native paid web: APIs that price themselves, and agents that pay them autonomously without an API key handshake or a billing portal.

## Prerequisites

- Python 3.12+
- `pip install sardis httpx`
- A Sardis sandbox API key in `SARDIS_API_KEY`
- An x402-enabled endpoint (Coinbase publishes test endpoints at `x402.org`)

## Step 1: What x402 actually is

HTTP 402 has been reserved in the spec since 1997 — "Payment Required" — and stayed unused for two decades. Coinbase's [x402](https://x402.org) spec finally gave it a payload: when an endpoint wants money, it returns `402` with an `X-Payment-Challenge` header describing the amount, the recipient, the accepted token, and a nonce. The client pays, attaches the proof in `X-Payment` on the retry, and the server validates and serves the response.

Why this matters for agents: there's no SDK to integrate per provider, no key to provision, no rate-limit dance. Discovery, pricing, and payment all happen in HTTP headers the agent can read. An agent can find an endpoint it has never seen before and pay for it without a human in the loop.

## Step 2: The naive call (and the 402)

Let's start by hitting a paid endpoint with plain `httpx`:

```python
import httpx

resp = httpx.get("https://api.example.com/premium/data")
print(resp.status_code, resp.headers.get("x-payment-challenge"))
```

You'll see `402` and a JSON-encoded challenge in the header. The challenge typically contains:

```json
{
  "amount": "0.05",
  "currency": "USDC",
  "chain": "base",
  "pay_to": "0xabc...",
  "nonce": "9f2c...",
  "expires_at": "2026-05-23T20:11:00Z"
}
```

A naive implementation parses this, calls a wallet, gets a tx hash, and constructs the proof header. The naive implementation is also where every x402 client has historically broken — chain selection, fee estimation, replay-nonce handling, and signature shape are easy to get subtly wrong.

## Step 3: Let Sardis settle the challenge

Sardis's `pay.execute` already speaks the x402 settlement shape. You hand it the challenge fields, it returns a settlement receipt with the proof header pre-built:

```python
import json, httpx, os
from sardis import Sardis

client = Sardis(
    api_key=os.environ["SARDIS_API_KEY"],
    base_url="https://sandbox.api.sardis.sh",
)

def pay_x402(challenge: dict) -> str:
    receipt = client.pay.execute(
        to=challenge["pay_to"],
        amount=challenge["amount"],
        currency=challenge["currency"],
        chain=challenge.get("chain"),
    )
    return receipt["tx_hash"]
```

`chain` is passed through explicitly because x402 challenges specify the rail; auto-routing across chains would defeat the merchant's settlement preference. Iron rule: if the merchant names the chain, the merchant gets the chain.

## Step 4: Wire it into a retry loop

The full flow — call, catch 402, pay, retry — is short enough to keep in a helper:

```python
def fetch_paid(url: str) -> httpx.Response:
    resp = httpx.get(url)
    if resp.status_code != 402:
        return resp

    challenge = json.loads(resp.headers["x-payment-challenge"])
    tx_hash = pay_x402(challenge)

    proof = json.dumps({
        "tx_hash": tx_hash,
        "nonce": challenge["nonce"],
        "chain": challenge["chain"],
    })
    return httpx.get(url, headers={"x-payment": proof})
```

Three details that bite people:

- **Re-read the challenge on every 402.** Nonces are single-use; you cannot cache a paid-for proof and replay it. The whole point of the nonce is to make replay impossible.
- **Set a per-call spending cap.** Always wrap `fetch_paid` so an attacker who controls a URL can't make you pay an arbitrary amount. A simple `if float(challenge["amount"]) > MAX_PER_CALL: raise` in front of `pay_x402` is enough.
- **Don't follow redirects through 402.** Disable `httpx` auto-redirects for paid calls so you can inspect each hop's challenge yourself.

## Step 5: Wrap with a budget mandate

For an agent doing many x402 calls in a session, attach a mandate so the *total* spend is capped regardless of how many endpoints it hits:

```python
intent = client.payments.create_intent_mandate(
    agent_id="research-agent",
    max_amount_minor=2_000_000,  # $2.00 across all x402 calls
    allowed_categories=["api", "ai_api"],
    expires_in_seconds=600,
)

def pay_x402(challenge: dict) -> str:
    receipt = client.pay.execute(
        to=challenge["pay_to"],
        amount=challenge["amount"],
        currency=challenge["currency"],
        chain=challenge.get("chain"),
        mandate_id=intent["id"],
    )
    return receipt["tx_hash"]
```

With the mandate attached, the eleventh $0.20 call against a $2.00 budget is rejected server-side with `amount_exceeded` — the agent literally cannot overspend, even if the calling code has a bug.

## Final: full working example

```python
import json, os, httpx
from sardis import Sardis

MAX_PER_CALL = 1.00  # USD ceiling per single 402 challenge

def main() -> None:
    sardis = Sardis(
        api_key=os.environ["SARDIS_API_KEY"],
        base_url="https://sandbox.api.sardis.sh",
    )

    intent = sardis.payments.create_intent_mandate(
        agent_id="x402-demo",
        max_amount_minor=2_000_000,
        allowed_categories=["api", "ai_api"],
        expires_in_seconds=600,
    )

    def fetch_paid(url: str) -> httpx.Response:
        resp = httpx.get(url, follow_redirects=False)
        if resp.status_code != 402:
            return resp

        challenge = json.loads(resp.headers["x-payment-challenge"])
        if float(challenge["amount"]) > MAX_PER_CALL:
            raise RuntimeError(f"challenge too expensive: {challenge['amount']}")

        receipt = sardis.pay.execute(
            to=challenge["pay_to"],
            amount=challenge["amount"],
            currency=challenge["currency"],
            chain=challenge.get("chain"),
            mandate_id=intent["id"],
        )
        proof = json.dumps({
            "tx_hash": receipt["tx_hash"],
            "nonce": challenge["nonce"],
            "chain": challenge["chain"],
        })
        return httpx.get(url, headers={"x-payment": proof})

    body = fetch_paid("https://x402.org/sandbox/echo").text
    print(body)

if __name__ == "__main__":
    main()
```

Swap in any x402-enabled URL — the loop is the same.
